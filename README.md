# TOPIK 모의시험 및 취약 유형 기반 학습 추천 시스템

## 프로젝트 개요

TOPIK 시험 대비를 위한 모의시험 웹 서비스를 기획하고, Node.js 기반 프로토타입을 구현한 프로젝트입니다.

사용자가 모의시험을 응시하면 정답/오답을 판별하고, 틀린 문제의 유형을 분석하여 자주 틀리는 유형과 유사한 문제를 다시 노출하는 **취약 유형 기반 학습 추천 구조**를 목표로 했습니다.

이 프로젝트의 핵심은 단순한 문제 풀이 서비스가 아니라, TOPIK 시험의 문제 번호별 고정 출제 유형을 활용하여 학습자의 취약 유형을 추적하고 반복 학습으로 연결하는 피드백 루프를 구성한 것입니다.

## 프로젝트 배경

일반적인 학습 추천 시스템은 사용자의 학습 이력이나 행동 데이터가 충분히 쌓여야 추천 정확도를 높일 수 있습니다.
하지만 초기 사용자에게는 데이터가 거의 없기 때문에, 추천 시스템에서 흔히 말하는 **Cold Start 문제**가 발생합니다.

이 프로젝트에서는 TOPIK 시험의 구조적 특징을 활용해 이 문제를 해결하려 했습니다.

TOPIK은 1번부터 50번까지 문제 번호별로 출제되는 유형이 비교적 고정되어 있습니다.
따라서 학습자 데이터가 충분히 쌓이지 않은 초기 상태에서도, 문제 번호와 문제 유형을 기준으로 학습 방향을 제시할 수 있다고 판단했습니다.

## 핵심 아이디어

TOPIK 문제는 번호별로 요구하는 능력과 출제 유형이 반복됩니다.

예를 들어 특정 번호 구간은 빈칸 추론, 주제 파악, 내용 일치, 순서 배열 등으로 유형이 고정되는 경향이 있습니다.
이 특성을 활용하면 사용자가 어떤 문제를 틀렸는지 확인하는 것만으로도, 어떤 유형에 약한지 추정할 수 있습니다.

```text
모의시험 응시
→ 정답/오답 판별
→ 틀린 문제 번호 확인
→ 문제 번호 기반 유형 분류
→ 자주 틀리는 유형 집계
→ 동일·유사 유형 문제 노출
→ 취약 유형 반복 학습
```

## 문제 코드 체계

문제는 시험 종류, 회차, 문제 번호, 유형을 구분할 수 있도록 코드 체계를 설계했습니다.

예시:

```text
010640402
```

의미:

```text
01  064  04  02
│   │    │   │
│   │    │   └─ 문제 유형 코드
│   │    └──── 문제 번호
│   └───────── 시험 회차
└───────────── TOPIK 구분
```

예를 들어 `010640402`는 다음과 같이 해석할 수 있습니다.

```text
TOPIK 1
64회
04번 문제
02유형
```

이 구조를 통해 문제를 단순히 저장하는 것이 아니라, 시험 회차와 문제 번호, 유형 정보를 함께 관리할 수 있도록 설계했습니다.

## 문제 유형 매핑 예시

문제 번호에 따라 다음과 같이 유형 타이틀을 자동으로 매핑했습니다.

```text
1~2번   → ( )에 들어갈 가장 알맞은 것을 고르십시오.
3~4번   → 밑줄 친 부분과 의미가 비슷한 것을 고르십시오.
5~8번   → 무엇에 대한 글인지 고르십시오.
9~12번  → 글 또는 그래프의 내용과 같은 것을 고르십시오.
13~15번 → 문장 순서 배열
16~18번 → 빈칸에 들어갈 내용 고르기
19~50번 → 독해 지문 기반 문제
```

이 구조를 통해 사용자가 틀린 문제 번호만 확인해도, 어떤 유형에서 약한지 추정할 수 있도록 했습니다.

## 기술적으로 의미 있었던 점

이 프로젝트의 핵심은 단순한 CRUD 웹 서비스가 아니라, 시험 구조 자체를 추천 시스템의 기준으로 활용했다는 점입니다.

일반적인 추천 시스템은 사용자의 누적 데이터가 부족하면 개인화 추천이 어렵습니다.
하지만 TOPIK은 문제 번호별 출제 유형이 반복되기 때문에, 초기 데이터가 부족한 상태에서도 문제 번호와 유형 정보를 기반으로 추천을 시작할 수 있습니다.

즉, 사용자 행동 데이터가 많지 않아도 문제 번호와 문제 유형 정보를 기준으로 기본 추천을 제공하고, 이후 오답 기록이 누적되면 자주 틀리는 유형을 중심으로 추천 정확도를 높이는 구조를 설계했습니다.

## 개선 방향

향후 개선한다면 다음과 같은 구조로 확장할 수 있습니다.

```text
사용자 응시 결과 누적
→ 문제별 정오답 데이터 저장
→ 문제 번호 기반 유형 분류
→ 유형별 오답률 계산
→ 취약 유형 Top N 추출
→ 동일 유형 문제 우선 추천
→ 학습 후 재응시 결과 비교
→ 추천 효과 검증
```

또한 단순히 틀린 문제를 다시 보여주는 방식이 아니라, 다음과 같은 추천 전략으로 확장할 수 있습니다.

* 같은 유형의 쉬운 문제부터 재노출
* 같은 유형의 다른 회차 문제 추천
* 오답률이 높은 유형을 우선순위로 정렬
* 일정 횟수 이상 반복 오답이 발생한 유형 별도 표시
* 유형별 학습법 또는 해설 콘텐츠 연결
* 학습 전/후 오답률 변화 시각화

## 프로젝트를 통해 얻은 점

이 프로젝트를 통해 웹 서비스 구조, 데이터베이스 연동, 세션 기반 사용자 관리, 시험 문제 데이터 모델링, 취약 유형 기반 추천 시스템 설계를 경험했습니다.

또한 추천 시스템이 반드시 복잡한 머신러닝 모델에서만 출발하는 것이 아니라, 문제의 구조와 도메인 규칙을 잘 이해하면 규칙 기반 추천으로도 충분히 시작할 수 있다는 점을 배웠습니다.

가장 중요한 경험은 아이디어 자체가 좋아도 실제 서비스로 완성하려면 데이터 구축, UI, DB 설계, 사용자 흐름, 추천 검증까지 모두 필요하다는 점이었습니다.
이 프로젝트는 완성도 높은 서비스로 끝나지는 못했지만, 문제를 정의하고 추천 구조를 설계하고 핵심 흐름을 구현해본 의미 있는 시도였습니다.

## 포트폴리오 요약

Node.js, Express, EJS, MySQL을 사용해 TOPIK 모의시험 웹 서비스 프로토타입을 구현했습니다.
사용자가 모의시험을 응시하면 정답/오답을 판별하고, 틀린 문제의 유형을 분석해 자주 틀리는 유형과 유사한 문제를 다시 노출하는 취약 유형 기반 추천 흐름을 설계했습니다.

TOPIK 문제 번호별 고정 출제 유형을 활용해 Cold Start 상황에서도 동작 가능한 규칙 기반 추천 구조를 만들었으며, 모의시험 응시, 오답 기록, 문제 번호 기반 유형 분류까지 핵심 흐름을 구현했습니다.
프로젝트 전체를 완성도 높은 서비스 수준까지 마무리하지는 못했지만, 해당 아이디어를 긍정적으로 평가받아 관련 포럼에 참석했습니다.
